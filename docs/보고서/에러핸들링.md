# 에러 핸들링

고정 파이프라인(OpenGL fixed-function) 기반 Keroro 렌더링에서 마주친 문제들과 해결 과정을 정리한다.
공통 형식: **증상 → 원인 → 해결(코드 예제)**. (현재 코드 기준)

---

## 1. 모자가 배경으로 뚫리고, 뒷면 텍스처가 앞면을 덮음

### 증상
- `KeroroHat`의 한쪽이 **뒷배경으로 뚫려** 보임.
- (깊이 쓰기를 끄는 임시 처치를 했을 땐) 반대로 **가려져야 할 뒷면이 앞면 위로 덮어씌워짐**.

### 원인 (곁가지에 속지 말 것)
처음엔 "렌더 순서·좌우 비대칭" 문제로 보였으나, **모델(회전 곡면)은 좌우 대칭이고 법선·winding도 정상**(수치 검증 완료)이다. 진짜 원인은 하나:

> **모자 텍스처가 알파 컷아웃(불투명 모자 + 투명 배경)인데, blend만 켜면 투명 텍셀(α≈0)도 깊이를 기록한다.**

- 가까운 면의 투명 텍셀이 깊이를 써버림 → 뒤쪽 불투명 면/배경이 깊이 테스트에 탈락 → **뻥 뚫림**.
- 이를 피하려 깊이 쓰기를 통째로 끄면 → 깊이 정렬이 사라져 그리기 순서대로 **뒷면이 앞면을 덮음**.
- 즉 `glDepthMask` 조작·2-패스·좌우 분기는 전부 곁가지였다.

### 해결
**투명 텍셀을 알파 테스트로 폐기**한다. 불투명 면만 깊이를 쓰므로 **일반 깊이 테스트가 앞면이 뒷면을 가리도록** 자동 처리 → Rotate·좌우·그리기 순서와 무관하게 정합. 단일 패스, 깊이 쓰기는 기본값 ON 유지.

```cpp
// src/model_imp.h — KeroroHat::Draw()
glEnable(GL_ALPHA_TEST);
glAlphaFunc(GL_GREATER, 0.9f); // α>0.9 텍셀만 통과(불투명 면만 깊이 기록)
ParametricGeometry::Draw();
glDisable(GL_ALPHA_TEST);
```

> 곁가지였던 시도: 모자 뒷면이 밋밋해 한때 `glLightModeli(GL_LIGHT_MODEL_TWO_SIDE, GL_TRUE)`로 양면 조명을 켰다. 하지만 알파 컷아웃으로 모자를 **불투명·앞면만 보이는** 형태로 만들자 양면 조명이 불필요해졌고(뒷면이 안 보임), 켜두면 오히려 뒷면 specular가 비쳐 하이라이트가 어긋났다. → **양면 조명 설정은 제거**(단면 조명으로 복귀).
>
> 교훈: 증상이 "좌우 비대칭/순서 의존"으로 보여도 **모델이 올바르면 깊이 테스트가 정합을 보장**한다. 정렬·depth-mask 트릭 전에 "투명 텍셀이 깊이를 쓰는가"부터 의심할 것.

## 2. U/V 범위를 반전하면 법선이 뒤집힘 (음영/하이라이트가 엉뚱한 면에)

### 증상
UI에서 파라메트릭 곡면의 `U_START/U_END` 또는 `V_START/V_END`를 드래그해 **범위를 반전**(시작 > 끝)하면, 음영이 반대 면에 찍힘.

### 원인
법선을 finite-difference로 구할 때 **양 방향 공통 스칼라 eps**를 썼다.

```cpp
const double eps = std::min(du, dv) * 0.01;          // 부호가 섞인 공통 스텝
normal = normalize(cross(P(u+eps,v)-P, P(u,v+eps)-P));
// = normalize(eps² · (∂P/∂u × ∂P/∂v))  → eps²>0 이라 부호가 소거됨
```

- 폴리곤 winding이 만드는 앞면 법선은 `du·dv·(∂P/∂u × ∂P/∂v)`.
- 공통 eps 법선은 부호가 `eps²`로 소거돼 **항상 `+(∂P/∂u × ∂P/∂v)`** 고정.
- 따라서 `du·dv < 0`(U나 V 중 **한쪽만** 반전)일 때 둘이 **반대 방향** → 법선이 앞면의 반대쪽을 가리킴.

> 수치 검증(정점 4096개): 한쪽 범위만 반전 시 **전 정점**의 법선이 winding과 반대였음.

### 해결
각 방향에 **자기 격자 간격에 비례하고 부호를 보존하는** 스텝을 쓴다 → `cross ∝ du·dv·(∂P/∂u × ∂P/∂v)`가 되어 winding과 항상 일치.

```cpp
// src/model.cpp — ParametricGeometry::build()
const double epsU = du * 0.01; // 부호 보존
const double epsV = dv * 0.01;
const glm::vec4 pu = SurfaceFunction(u + epsU, v);
const glm::vec4 pv = SurfaceFunction(u, v + epsV);
// 정상 범위에서는 결과 동일(기존 장면 변화 없음), 반전 시에도 정합 보장.
```

부수 효과: 큰 범위(예: u∈[0,2π]) 방향에 과도하게 작은 스텝을 강요하던 float 자릿수 손실도 완화.

## 3. 모델 id 1:1 불변식 깨짐 (손 모델 중복 생성)

### 증상
손(`KeroroHand`)을 추가하면 같은 id로 모델이 **2개** 생겨 id가 어긋나고, UI 라벨 생성에서 인덱스 초과 접근(크래시 위험)·저장 파일 오염 발생.

### 원인
`AddModel`은 끝에서 메타를 **1개만** 등록하는데, 손 케이스만 `renderer.AddModel(...)`을 **두 번**(좌·우) 호출 → "모델 수 ≠ 메타 수".

### 해결
"**한 번 호출 = 모델 1개 = id 1개**" 불변식을 복원. 좌/우는 `variant` 인자로 기본 포즈만 선택(영속 상태는 transform).

```cpp
// apps/main.cpp
bool AddModel(ModelType type, int id, int variant = 0); // variant: 좌=0 / 우=1
// 손 케이스: 한 번에 하나만 생성
renderer.AddModel(make_unique<KeroroHand>(tex, variant));
// 기본 씬에서 좌·우를 각각 등록
AddModel(ModelType::KeroroHand, id++, 0); // 왼손
AddModel(ModelType::KeroroHand, id++, 1); // 오른손
```

## 요약

| # | 증상 | 핵심 원인 | 해결 |
|---|---|---|---|
| 1 | 모자 면 뚫림 / 뒷면이 앞면 덮음 | 투명 텍셀이 깊이를 기록 | 알파 테스트로 투명 텍셀 폐기(`α>0.9`) |
| 2 | U/V 반전 시 음영 반대 | 공통 eps가 부호 소거 | 방향별 부호보존 eps |
| 3 | 손 id 중복/크래시 | 1호출 2모델 | 1호출 1모델 + variant |

> 참고: 모자 뒷면 조명을 위해 한때 켰던 양면 조명(`GL_LIGHT_MODEL_TWO_SIDE`)은 알파 컷아웃 도입 후 제거되어 현재 코드에는 없다.
